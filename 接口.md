- 最重要思想：规则
- Java中的类只支持单继承，所以创建了接口实现==多继承（实际是多实现）==
- 代码中想要声明规则，就使用抽象类
- 对于只是单纯在声明抽象方法的类，可以改写成接口“interface”
1. 定义格式
	1. public interface 接口名{}
	2. 和抽象类一样，也不允许实例化
	3. 接口和类之间是实现关系
		1. 格式：public class 类名 implements 接口名{}
	4. 接口的实现类
		1. 要么重写所有抽象方法
		2. 要么抽象实现类
2. 接口的成员特点
	1. 成员变量
		1. 可以继承成员变量使用，但接口的成员变量是public static final这三个关键字修饰的，不可再被实现类更改，而且只能是常量。（被定义的规则不能被修改）
	2. 成员方法
		1. 接口中不能定义普通方法，必须是抽象方法（默认会加，不用手动添加public 和abstract这两个关键字，也不能被实现类修改）
	3. 构造方法
		1. 接口中没有构造方法
		2. 实现类中的构造方法的默认super（）在访问object类（所有类都继承object类）
3. 类和接口的关系
	1. 类之间是继承的关系，只能单继承不能多继承
	2. 类与接口之间是实现的关系，既能单实现也能多实现，一个类可以实现多个接口
	3. 若子类继承的父类中有一个和实现的类中同名的方法，则不用重写，因为父类的方法相当于重写过了
	4. 接口与接口之间
		1. 继承关系可以单继承也可以多继承
		2.  实现类需要把接口继承的接口全部实现
