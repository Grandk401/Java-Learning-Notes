2. 算数运算符
	1. %：取余操作，获得的是两个数据做除法的余数==整数相除只能得到整数，没有四舍五入的概念，在运算中加入一个浮点数==

>**案例：数值拆分，键盘录入一个三位数，将其拆分为百位、十位、个位
* 使用scanner录入三位数，数值%10得到个位，/10%10得到十位，/10/10%10得到百位,更多位以此递增
```Java
import java.util.Scanner;  
public class HelloWorld {  
    public static void main(String[] args) {  
        System.out.println("请输入一个三位数：");  
        Scanner sc = new Scanner(System.in);  
        int num = sc.nextInt();  
        int Ge = num % 10;  
        int Shi = num /10 % 10;  
        int Bai = num /10 /10 % 10;  
        System.out.println("个位数是：" + Ge);  
        System.out.println("十位数是：" + Shi);  
        System.out.println("百位数是：" + Bai);  
        sc.close();  
    }  
}
```

6. 类型转换
	1. 隐式转换：把一个取值范围小的变量赋值给一个取值范围大的变量
		1. 数据从小到大：byte-short/char-int-long-float-double（小数底层的二进制存储要比整数更加节省空间）
		2. 运算中的隐式转换：取值范围小的数值和取值范围大的数值进行运算时，小的会先提升为大的再进行运算，最后得到的结果也是取值范围大的数据类型
		3. byte、short、char三种数据在运算的时候都会先提升为int再进行运算，最后得到的结果也是int
	2. 强制转换
		1. 把一个取值范围大的变量转换成一个取值范围小的变量
		2. 格式：目标数据类型 变量名 = （目标数据类型）被强转的数据；double a = 12.3；int b = （int）a；
		3. 强制转换有可能损失精度
		4. 二进制的三种状态
			1. 原码：数据二进制的体现形式，最左侧（即==高位==）0代表正数，1代表负数，遇到负数运算可能会出现错误
			2. 反码：正数的反码和原码相同；负数的反码是对其原码逐位取反，符号位除外
			3. 补码：计算机的二进制运算时==只使用补码==。正数的补码与其原码相同；负数的补码是在其反码的末位+1 
			4. 二进制中先减一再取反和先取反再加一结果是一样的
